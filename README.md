# NULL//SHIFT — GitHub Pages P2P Edition

GitHub Pagesだけで公開できる、サーバーレス版です。ゲーム通信はWebRTCでプレイヤー同士を直接接続します。サーバー費用はかかりません。

## GitHub Pagesで公開

1. このフォルダ内のファイルをリポジトリ直下へアップロード
2. GitHubの `Settings` → `Pages`
3. `Deploy from a branch`、ブランチ `main`、フォルダ `/(root)` を選択
4. 表示された `https://ユーザー名.github.io/リポジトリ名/` を開く

## 接続方法

### ホスト
1. 「P2Pルームを作る」
2. 招待コードを相手へ送る
3. 相手から返された応答コードを貼り、「応答を適用」

### ゲスト
1. 「招待コードで参加」
2. ホストの招待コードを貼り、「応答コードを生成」
3. 出力された応答コードをホストへ返す
4. ホストが適用すると自動的に接続

招待コードは接続情報そのものなので長い文字列になります。WebRTCの仕様上、完全静的サイトで6桁コードだけを使うことはできません。

## 特徴と制限

- Node.jsや常駐サーバーは不要
- GitHub PagesのHTTPSで動作
- ゲームデータはP2P接続
- ボイスはホストと各ゲスト間で利用可能
- テキストチャットは全員へ中継
- NATや学校・企業ネットワークの設定によってはP2P接続できない場合があります
- 無料のSTUNのみ使用し、TURNサーバーは使用しません

ローカル確認は `index.html` の直接オープンではなく、`python -m http.server` などの静的サーバー、またはGitHub Pages上で行ってください。
